Cours GIT

Qu'est ce que GIT

Git est un outils de visionning et il permet d'avoir une tracabilité et historique de tous les fichiers du projet

Il permet également de travailler en équipe: Envoyer nos modification à nos collègues et récupérer les modifications des collègues.

Dans la tracabilité des fichiers GIT nous donnes toujours 3 infos: Qui, Quand, Pourquoi?

Installer et configurer GIT

url: https://git-scm.com/
git config --global user.email 'lindsay.fraynet@gmail.com'
git config --global user.name 'lindsay fraynet'

Les commandes de bases

Initialiser un projet:
git init
Voir le statut des fichiers:
git status
Ajouter TOUS LES FICHIERS
git add.
Ajouter des fichiers:
git add 'nom du fichier'
Commit (enregister) les fichiers ajoutés:
git commit
un commit nécessite toujours un message
Commit avec message:
git commit -m 'message'
Commit avec add + message (fonctionne sur les fichiers déjà add précédemment):
git commit -a -m 'message'

Correction des fichiers

Voir les différernts commit:
git log
Corriger les modifications sur un fichier (pas encore add):
git checkout 'nom du fichier'
Supprime un add:
git reset 'nom du fichier'
git reset . 'nom du fichier' (tout les fichiers depuis le dernier commit)
Supprimer le dernier commit:
git reset HEAD^ 'nom du fichier'
Supprimer les deux derniers commit:
git reset HEAD^^ 'nom du fichier'

La notion de branche

Une branche permet est une copie du projet. Elle permet de développer en paralèlle de nouvelle fonctionnalités. Lorsqu'on considère les nouveautés opérationnelles, on peut les rappatrier sur le version principale (master)

Voir les branches:
git branch
Créer une nouvelle branche:
git branch 'donner un nom'
Se placer sur une branche:
git checkout 'nom de la branche en question'
Voir les différences entre deux branches:
git diff 'nom de la branche' ..master
Apporter les modifs effectuées sur une branche dans master (il faut être sur master)
git checkout master
git merge 'nom de la branche'
Remplacer master par une branche:
git checkout master
git fetch . 'nom de la branche
Le fetch va littérallement remplacer master par la branche. Il s'agit donc d'une grosse modificaion dans le projet (ex: version 2)
